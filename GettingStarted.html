Getting Started

Thank you for trying out the Web Security Dojo!

Here you’ll find popular tools for testing web application security, along with some targets containing intentional vulnerabilities. Web Security Dojo is intended to be used as a training environment, and probably shouldn’t be used as a pentesting platform due to the vulnerable services included. For a project better suited to performing security assessments see Kali Linux.

To get started, fire up Firefox using the menu or icon above and connect to DVWA by clicking the DVWA link on the Firefox home page. Alternatively, just surf to http://dvwa.local/dvwa

DVWA has good help and multiple difficulty settings you can try dependent on your skill level.

Another good learning tool to try is WebGoat, which can be started by going to Applications → Targets → WebGoat NG Start. The main page will load in Firefox on startup. If you need to get back to the main page, it’s available in Bookmarks → Vulnerable Web Apps or on the Firefox home page like the rest of the targets.

After you poke around manually for a while, you will probably want to start using a intercept tool like Burp Proxy to view and modify the traffic. Start with Applications → Tools → Burp Suite Free and then use MM3 Proxy Switch via the pull down menu next to the URL bar in Firefox to select Burp Suite. Then all your traffic will flow through Burp Proxy. To have your traffic flow normally, just click the main MM3 button to turn off the proxy (it will change color from dark red [on] to grey [off]).

These exercises show you the basics you need to start exploring the dojo on your own. Other hints can be found in the Applications → Documentation section, which contains some useful free documentation on web application testing and some of the specific target documentation. Also, we have a video page on Youtube with a few video tutorials. More to come soon.

We highly recommend you use the Zim Wiki or one of the text editors to take notes on your tests and findings. Taking good notes is an essential skill for security testing.

If you wish to make changes or update the system, the username and password are both dojo.

There is no root user, if you need to do something as root, use sudo and the dojo password.


Web security dojo is an open source project, and we welcome contributions or collaboration with other projects. Our target and tool installs are completely scripted in bash.
